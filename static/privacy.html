Privacy Policy

Effective date: March 2, 2026

1. Overview

CAAMS Enterprise ("the Software") is a self-hosted audit and compliance management platform. This Privacy Policy describes how data is collected, stored, and used when you or your organization deploys and operates the Software.

Because CAAMS Enterprise is deployed on infrastructure you control, your organization is the data controller for all personal data stored in the system. This policy describes the Software's data practices; your organization's own privacy policies and data processing agreements govern your end users' rights.

2. Data We Collect

The following categories of data are stored in the database and logs when you use the Software:

  • Account data: usernames, email addresses, hashed passwords (PBKDF2-SHA256, 600,000 iterations), account roles, and account creation timestamps.
  • Authentication data: TOTP (MFA) secrets stored encrypted at rest, OIDC subject identifiers and provider information, session tokens (stored as hashed values), and token version counters used for revocation.
  • Audit and activity data: an immutable audit log recording user actions (assessments created/updated, evidence uploaded, logins, permission changes) including the acting username, timestamp, IP address, and action detail.
  • Assessment and compliance data: assessment names, framework selections, control responses, evidence descriptions, file references, findings, RFIs, and comments entered by users.
  • Application logs: structured JSON logs written to logs/app.log and emitted to stdout, including log level, logger name, message, and any additional fields provided at call sites (which may include usernames and IP addresses).

3. How Data Is Used

Data collected by the Software is used exclusively to:

  • Authenticate users and enforce role-based access control.
  • Provide audit and compliance management features to authorized users.
  • Maintain an immutable audit trail of user actions for compliance and security purposes.
  • Diagnose application errors and monitor system health.

CAAMS Enterprise does not transmit any user data to external services, analytics platforms, or third parties unless your deployment explicitly integrates with an external identity provider (OIDC) or SIEM system of your choosing.

4. Data Storage and Retention

All data is stored in the PostgreSQL database and log files on your infrastructure. The Software does not enforce a data retention period by default; your organization is responsible for defining and implementing retention policies appropriate to your compliance obligations.

Log files are rotated at 10 MB with five backup files retained (approximately 50 MB total per log stream). Older log files are deleted automatically by the log rotation process.

5. Data Security

The Software implements the following security controls:

  • Passwords hashed with PBKDF2-SHA256 at 600,000 iterations.
  • Session tokens stored as hashed values; raw tokens are never persisted.
  • MFA secrets encrypted at rest using AES-GCM via the CAAMS_SECRET_KEY environment variable.
  • All API endpoints require authentication except login, setup, and public share routes.
  • Rate limiting applied to all authentication and sensitive endpoints.
  • Security headers (X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy) set on all responses.

6. Third-Party Integrations

The Software may integrate with the following external services, all of which are configured and controlled by your organization:

  • OIDC Identity Provider: if SSO is enabled, usernames, email addresses, and OIDC subject identifiers are exchanged with your configured IdP during authentication.
  • SIEM / Log Management: if your infrastructure routes log output to an external SIEM, the structured JSON logs described in Section 2 will be ingested by that system under your organization's data processing policies.
  • SMTP: if email notifications are enabled, user email addresses are transmitted to your configured SMTP server for notification delivery.

7. Individual Rights

If your organization is subject to GDPR, CCPA, or similar regulations, you are responsible as the data controller for handling data subject requests (access, deletion, portability, correction). The Software provides admin-accessible user management and audit log export features that can assist with fulfilling these requests.

8. Changes to This Policy

This policy may be updated as the Software evolves. The effective date at the top of this page reflects the most recent revision. Material changes will be noted in the release changelog.
